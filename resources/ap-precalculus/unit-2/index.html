2.1

Change in Arithmetic and Geometric Sequences

Sequences are functions whose inputs are usually whole-number indices. The strongest solutions identify the invariant that drives the sequence, state the domain, and coordinate at least two representations rather than relying on visual pattern recognition alone.

2.2

Change in Linear and Exponential Functions

Linear and exponential functions can both increase or decrease, but they organize change differently. A linear model adds the same amount over equal input intervals; an exponential model multiplies by the same factor. Strong comparisons focus on the invariant and on what the parameters mean in context.

2.3

Exponential Functions

An exponential function is controlled by a starting scale, a multiplicative base, and possible transformations. A complete analysis names the horizontal asymptote, specifies whether the graph approaches it from above or below, and distinguishes growth or decay from increasing or decreasing behavior after reflections.

2.4

Exponential Function Manipulation

Manipulating exponential functions is not cosmetic: a useful form can reveal an initial value, a factor for a new time unit, or a graph transformation. Every rewrite should preserve the function and make one feature easier to interpret.

Exponential Function Context and Data Modeling

Exponential modeling begins with a multiplicative mechanism, not merely a curved graph. A strong model identifies the reference value, the factor and its interval, the input domain, and the assumptions that make repeated proportional change reasonable.

2.6

Competing Function Model Validation

Model selection is an argument supported by structure, not a contest to find the largest correlation number. Strong validation combines a plausible mechanism, a well-scaled residual analysis, comparison with competing models, and an explicit statement of the intended domain.

2.7

Composition of Functions

A composition represents a process applied to the output of another process. The inside function acts first. A complete solution tracks the intermediate value, verifies that it belongs to the outer function's domain, and interprets the order in context.

2.8

Inverse Functions

An inverse function reverses a one-to-one relationship. The domain of the original becomes the range of the inverse, and the range of the original becomes the inverse domain. Restrictions are part of the definition, not optional details.

2.9

Logarithmic Expressions

A logarithm answers an exponent question. Every valid logarithm has a positive argument, a positive base different from $1$, and a real output. Exact evaluation relies on recognizing powers; approximation uses change of base or calculator functions while preserving domain restrictions.

2.10

Inverses of Exponential Functions

Logarithmic functions arise by reversing exponential functions. The horizontal asymptote of the exponential becomes a vertical asymptote of the logarithm, and the exponential range becomes the logarithm domain. A transformed inverse must preserve every shift, scale, and restriction.

2.11

Logarithmic Functions

A logarithmic function compresses multiplicative input change into additive output change. Its vertical asymptote marks the boundary of the domain, and transformations must be interpreted through the argument as well as the outside coefficient.

2.12

Logarithmic Function Manipulation

Logarithmic properties transform multiplication into addition, division into subtraction, and powers into coefficients. These identities are valid only when every logarithm involved has a positive argument. A correct rewrite includes the same domain as the original expression.

2.13

Exponential and Logarithmic Equations and Inequalities

Exponential and logarithmic equations are solved by exposing the inverse structure. Exact solutions often use common bases or logarithms; approximate solutions use technology only after a correct algebraic setup. Logarithmic equations require a final domain check because algebraic manipulation can create invalid candidates.

2.14

Logarithmic Function Context and Data Modeling

Logarithmic models are appropriate when equal multiplicative changes in input produce approximately equal additive changes in output. They often describe diminishing returns or scale measurements, but their restricted domains and slow long-run growth must be interpreted in context.

2.15

Semi-log Plots

A semi-log plot makes exponential data linear by plotting the input against the logarithm of a positive output. Equal vertical distances represent equal ratios, not equal differences. The line's intercept determines the exponential coefficient, and its slope determines the growth or decay factor.
